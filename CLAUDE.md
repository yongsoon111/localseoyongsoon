## 개발 워크플로우 (자동)

### 기능 개발 요청 시
1. 요청 분석 → 독립된 기능들로 분해
2. **자동 판단**:
   - 독립 가능하면 → Task 병렬 실행
   - 순서 필요하면 → 순차 실행
3. 각 Task 완료 시 자동 머지

### Task 분리 기준
- 파일이 겹치지 않으면 → 병렬
- 같은 파일 수정 필요 → 순차
- 의존성 있으면 → 순차

### 자동 머지 규칙
- 테스트 통과 시 자동 머지
- 실패 시 사람에게 보고
- 충돌 발생 시 중단 후 보고

---

## 기능 구현 시 자동 분할

새 기능 요청 시 7개 Task로 분리:
1. Component (메인 컴포넌트)
2. Styles (CSS)
3. Tests (테스트)
4. Types (타입 정의)
5. Hooks (커스텀 훅)
6. Integration (라우팅, import)
7. Config (설정 파일)

각 Task 독립 실행 → 완료 순서대로 자동 머지

---

## 서브에이전트 위임 규칙

작업 타입별 자동 위임:
- 코드 작성 → developer
- 코드 완성 후 → reviewer (자동)
- 테스트 실패 → tester (자동)
- 보안 이슈 → security-auditor (자동)
